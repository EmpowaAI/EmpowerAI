In plain language

What does 100/100 actually mean?

Think of open source readiness the same way you’d think about a property inspection before listing a house. A score of 100/100 means every room is in order: the legal paperwork is signed, the structure is sound, the instructions for visitors are clear, and there’s a welcome mat at the door.

For EmpowerAI, this means: any skilled developer in the world can find the project, understand how to contribute, and trust that their contribution will be properly reviewed before it becomes part of the product. Nothing is hidden, nothing is missing. The project is professionally presented.

This is the standard that projects like VS Code, React, and Django hold themselves to. We now hold ourselves to the same standard.

Completed this sprint

Eight things we put in place

v1.0.0 Release published
The first official version is live on GitHub. Anyone referencing EmpowerAI now has a stable version number they can cite and build on.
Good First Issue labels
Specific issues on the tracker have been marked so that newcomers to open source can find genuinely approachable tasks to work on.
Code reviewer updated
The CODEOWNERS file now reflects the current team. Every proposed change to the codebase requires Nicolette’s explicit approval before it is accepted.
59 automated tests
Tests now verify that security, encryption, and core error handling behave correctly — and flag any regression immediately.
Security policy (SECURITY.md)
Tells security researchers exactly how to report vulnerabilities privately, so issues are disclosed responsibly — not posted publicly.
MIT Licence
The legal framework is in place. The MIT licence is the most permissive and widely trusted in open source — it allows others to use and build on EmpowerAI freely.
Contribution guide (CONTRIBUTING.md)
Step-by-step instructions for any developer who wants to contribute — from forking the repo to submitting a pull request.
Code of Conduct
Sets the community standards. Every serious open source project has one — it tells contributors what behaviour is expected and how violations are handled.

What comes next

The project is now discoverable worldwide

1
GitHub surfaces EmpowerAI to first-time contributors
GitHub runs a global discovery feed for developers looking for their first open source contribution. Issues tagged “good first issue” appear in this feed. EmpowerAI is now on that map — developers browsing for South African projects, career-tech projects, or AI projects can find and contribute to it without any outreach from us.
2
v1.0.0 can be cited, shared, and built on
Journalists, researchers, and other developers can now reference EmpowerAI at a specific, stable version. The release page on GitHub shows the full change history and makes the project look like a professional, shipped product — because it is.
3
Security researchers can report responsibly
The security policy means any developer who discovers a vulnerability now has a clear, private channel to report it directly to the team. This is how large companies like Google and Microsoft handle this — and EmpowerAI now does too.
4
20 new issues filed for next development sprint
A full codebase audit was completed this session. 20 new issues are logged on GitHub covering security hardening, bug fixes, and feature enhancements — giving contributors a well-organised backlog to work from.
